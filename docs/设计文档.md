# XM4 状态栏控制 — 设计文档

> 创建日期：2026-06-22
> 状态：待确认（工具链安装中）

## 一、项目目标

在 Windows 11 上实现一个驻留系统托盘（状态栏）的小程序，配合 Sony WH-1000XM4 耳机：

1. **托盘图标显示耳机连接状态 + 电量百分比**（tooltip 文字）。
2. **左键单击托盘图标 → 打开控制界面**（与源程序行为一致，`LeftClickCommand.Show()` → `MainWindow.ShowWindow()`），在界面内开关降噪/调环境声等级（上游已实现）。
3. 开机自启动（可选，复用上游 AutoRun 机制）。

非目标：不改源程序交互逻辑；左键不直接切换 NC（与上游一致才打开界面）；不重做设置页 UI；不实现均衡器/Sound Position 等（上游已有，不破坏）。

**改造增量仅「电量显示」**：托盘 tooltip + 控制界面可选加一行电量。NC/音量配置完全复用上游。

## 二、可行性结论（已调研）

### 2.1 降噪（NC）切换
- 上游 `WinUI-SonyHeadphonesClient` 已实现：`Headphones::setAmbientSoundControl(bool)` → `setChanges()` → `CommandSerializer::serializeNcAndAsmSetting` 发 `NCASM_SET_PARAM (0x68)` 命令。
- `CommandSerializer.cpp` 内 `MAX_STEPS_WH_1000_XM4 = 21`，已针对 XM4 调校。
- 连接走 RFCOMM 到 Sony 私有服务 UUID `96CC203E-5068-46ad-B32D-E316F5E069BA`，`WindowsBluetoothConnector` 已实现。
- 结论：**已可直接用，无需改造协议层。**

### 2.2 电量查询（关键调研成果）
来源：反编译的 Sony 官方 Songpal 安卓应用（`egaebel/sony-headphones-hack`），与上游 `Constants.h` 的 `NCASM_SET_PARAM=0x68` 交叉验证一致，可信度高。

| 项 | 值 |
|---|---|
| 通道 DATA_TYPE | `0x0C` (DATA_MDR) — 与 NC 命令同通道 |
| 查询载荷（转义前） | `[0x10, 0x00]`（COMMON_GET_BATTERY_LEVEL + BatteryInquiredType.BATTERY） |
| 响应命令字节 | `0x11`（RET）/ `0x13`（NTFY 主动通知） |
| 电量百分比位置 | 响应载荷 **byte[2]**，范围 0–100 |
| 充电状态位置 | 响应载荷 byte[3]：0=未充电 / 1=充电中 / 0xF0=未知 |

帧封装逻辑（`packageDataForBt`，上游已有）：
```
0x3E | DATA_TYPE | seq | BE4字节长度 | 载荷 | 累加校验和 | 0x3C
              （中间整段对 60/61/62 做转义：sentry=0x3D）
```
结论：**复用现有帧封装，新增 1 个命令常量 + 1 个查询/解析函数即可。**

### 2.3 Windows WinRT 电池 API（备选）
`GattBatteryService.GetBatteryLevelPercentageAsync` 或 `DeviceInformation` 属性 `System.Devices.Aep.BatteryLevel`（=`{104EA399-...} 2`）。对 XM4 是否返回有效值未确证（XM4 是经典 BR/EDR 音频设备，未必暴露 BLE 0x180F）。**仅作兜底，不主用。**

## 三、技术方案

### 3.1 不重写，改造上游
基于 `reference/WinUI-SonyHeadphonesClient-upstream` 改造（C# WinUI 3 前端 + C++ DLL `SonyHeadphonesClientDll` 协议核心）。不另起纯 C# 项目——上游 NC 切换已成熟，重写协议层风险更高。

### 3.2 改造清单

**A. C++ DLL 侧（`SonyHeadphonesClientDll`）**

1. `Constants.h`：新增枚举
   - `COMMAND_TYPE::COMMON_GET_BATTERY_LEVEL = 0x10`
   - 响应：`COMMON_RET_BATTERY_LEVEL = 0x11`、`COMMON_NTFY_BATTERY_LEVEL = 0x13`
   - `BATTERY_INQUIRED_TYPE::BATTERY = 0`
2. `CommandSerializer.cpp/.h`：新增
   - `Buffer serializeBatteryLevelQuery()` → 返回 `[0x10, 0x00]`
3. `BluetoothWrapper`：新增 `sendCommandWithResponse(payload)` —— 发送后读取响应帧，返回去转义后的载荷 `Buffer`（现有 `_waitForAck` 只读一帧当 ACK 用，需扩展为返回载荷）。
4. `Headphones`：新增 `int queryBatteryLevel()` —— 发查询、解析 byte[2]，返回 -1 表示失败。
5. `dllmain.cpp`：新增导出 `int GetBatteryLevel()`。

**B. C# 前端侧（`SonyHeadphonesClient`）**

6. `API/Headphone.cs`：新增 P/Invoke `GetBatteryLevel()`。
7. 托盘图标轮询：连接成功后启动 `DispatcherTimer`（间隔 60s），调用 `GetBatteryLevel()`，更新：
   - 托盘图标 tooltip：`WH-1000XM4  85%` / `WH-1000XM4  未连接`
   - （可选）图标 overlay 数字角标
8. **保留源程序左键行为**：左键单击 → 打开控制界面（`MainWindow.ShowWindow()`，与上游 `LeftClickCommand` 一致），不改。开关降噪/调音量在上游控制界面完成，已实现。
9. 控制界面（SettingPage）加一行电量显示（可选增强）。
10. 连接断开时停止轮询、图标置灰。

### 3.3 风险与对策

| 风险 | 对策 |
|---|---|
| `_waitForAck` 把响应当 ACK 丢弃 | 新增 `sendCommandWithResponse` 区分 ACK/响应；NC 命令仍走旧路径 |
| 电量查询期间耳机可能并发推送 NC 状态通知 | 解析响应时按 COMMAND_TYPE 分流（0x11=电量，0x68=NC） |
| 单实例连接被官方 App 抢占 | 文档注明：使用本工具时关掉 Sony Headphones Connect 官方 App |
| 帧分片（>2048B） | 电量响应很小，不会触发；保留上游断言 |

## 六、踩坑记录（避免重蹈）

1. **VS Installer 命令行不支持 `--wait`**：传 `--wait` 直接 exit 87（ERROR_INVALID_PARAMETER）。用 `--passive`（显示进度条、无交互、装完自关）替代，**不要用** `--wait`。
2. **`--quiet` 会静默放弃**：winget 跑 VS 安装时 `--quiet` 导致「已放弃安装」，只注册了壳没装工作负载。务必用 `--passive`。
3. **winget 触发 UAC 需用户点「是」**：非交互 shell 无法替你点 UAC，1602=用户取消。VS 这类需提权的安装，在管理员终端直接跑 `setup.exe` 最稳。
4. **对已注册的 VS 实例用 `modify` 不用 `install`**：`install` 对已存在实例会 exit 87，正确动词是 `modify --installPath <path>`。
5. **vswhere 在安装进行中 `state=13`**：state=13 表示正在安装，此时 `-latest -property installationPath` 可能返回空，属正常，等装完。

```xml
<StackPanel Orientation="Horizontal">
  <ToggleSwitch x:Name="AmbientSoundControl" x:Uid="AmbientSoundControl" .../>
  <Button Click="ShowHelpAmbient" Background="Transparent" Padding="2">
      <TextBlock Text="❓" FontSize="14"/>
  </Button>
</StackPanel>
```
